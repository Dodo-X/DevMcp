🧹 数据清理

?🧹 数据清理
清理过期和无效数据，支持按范围执行：
• 全部 — 归档清理 + 软删除物理清除
• 旧日志 — 清理过期的 improvement_log / evolution_log
• 孤立数据 — 清理无关联的 task_queue 记录
点击"预览"可先查看将要清理的内容而不实际执行。

📦 数据归档

?📦 数据归档
将旧数据按生命周期分层处理，释放存储空间：
• 温归档(30-180天) — 压缩步骤详情，保留摘要
• 冷归档(>180天) — 迁移至归档表，依赖日报MD
• 深度清理(>365天) — 删除归档记录
• 日志清理(>90天) — 清除过期操作日志
⚠️ 归档前会自动校验日报MD是否已导出。
归档旧数据以释放空间

🗑️ 清理 Pending 任务

?🗑️ 清理 Pending 任务
清理超过指定时间仍未执行的 pending/queued 任务：
• 时间范围 — 清理多久之前的任务（默认 24h）
• 任务类型 — 可限定 step_analysis 或全部
• 预览模式 — 先查看将要清理的内容
• 级联清理 — step 失败时自动取消对应的 conversation_finalize
💡 适用于 20+ 任务堆积、超时雪崩后的手动清理。

📊 Pending 任务统计

?📊 Pending 任务统计

• 按任务类型分组统计
• 按时间分布（<1h / 1-6h / 6-24h / >24h）
💡 用于评估是否需要手动清理。
